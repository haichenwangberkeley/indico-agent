# Briefing Style

Briefings should be polished narrative prose, not extracted line lists.

Use short section headings when helpful, but write the body as coherent paragraphs. Do not use bullet lists for the user-facing briefing. The extracted text and keyword matches are source material only; they should be reorganized into grammatical statements that explain what happened, what the result is, what the editorial board reacted to, and what follow-up is needed.

For public result approval meetings, cover the following in prose:

Main result and scope. State the analysis topic, dataset, final state, and result status. Quote numerical values only when extraction preserves them clearly; otherwise say that the source deck contains the numerical values but they were not reliable in text extraction.

Editorial board reaction. Summarize the substantive review concerns, especially interpretation, modelling, uncertainty treatment, presentation clarity, and approval-readiness.

Minutes and action items. Convert reviewer questions and proposed replies into concrete follow-up statements. Prefer "The draft should..." or "The team plans to..." over raw question fragments.

Risk/readiness. End with a short judgment about whether the material reads as mature, blocked, or still requiring significant analysis work.
